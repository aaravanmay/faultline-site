Get started

Let's catch your first
silent failure.

Four small steps. By the end, faultline will be breaking your agent's tools on purpose — and telling you the moment it confidently does the wrong thing.

1

Name your project Created

A project groups the agents you test and holds your CI token. You can rename it later.

URL slug: support-agent
2

Install

Add faultline to the repo your agent lives in. Pure Python, zero config.

pip install faultline
3

Wrap your tools + write one invariant

Decorate the tools your agent calls, then state the one thing that must stay true. faultline corrupts each tool in turn and watches whether your agent notices.

from faultline import tool, chaos from faultline.faults import WrongNumber, StaleData, Timeout @tool # wrap each tool your agent calls def lookup_order(id): ... @tool def issue_refund(order, amount): ... # the one thing that must stay true, on broken data or not def oracle(answer, ctx): assert answer.refund <= ctx.order_total # never over-refund result = chaos(agent, task='refund my last order', oracle=oracle, faults=[WrongNumber('lookup_order'), StaleData('lookup_order'), Timeout('issue_refund')]) result.report()
No real side effects. Action tools like issue_refund are captured during a test — faultline records what the agent would do, it never actually fires them.
4

Run your first check

Run the suite and push the results up here. In CI, this is one step in your pipeline.

faultline run faultline_suite.py --push
# faultline reads this to know where to send results export FAULTLINE_TOKEN=flt_••••••••••••••••

The token tells faultline which project to report to. Add it to your CI secrets — don't commit it.
